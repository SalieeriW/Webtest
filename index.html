Basically CSS is uses to give the structure (HTML) some design. As we can see here.

When it comes to the declarations of the CSS, it can be at the beginning, inside head or externally inserted (a .css file) or directly inside the tag. The tag for css is "style" and it's sytaxs consists in the declaration of the target followed by "{"" and properties that we want to give to the target.

Example

Notice that in the declaration of a style it's possible to combine diferent targets. For example, we can target by class, by id, by tag or combining them. For example, we can associate classes with tags (the tags with the class "blabla" will have the propeties we asign). There's aswell other interesting declations.
  1. Direct child target: When we want to target nested tags that are a direct child of it's parent, we can use this type of targeting. It's systaxs is:

    tag1 > tag2: where tag2 is a direct child of tag1.

  2. Non-direct target:Similar to the previous one but including al the child, grandchilds of the parent. In another words, if tag2 is inside of tag1 somehow, it's targeted. It's sytaxs is:

    tag1 tag2: Where tag2 is inside of tag1.

In this sections we will be talking about the floating technique, which consists in moving boxes. In order to make boxes move, we need to commands, first and obvious, float and a direction (left o right). And clear, to make sure that nothing get covered.

When I say covered, we have to understand one thing: When we float a element, this element leave the orginal html flow, so other boxes may ignore him and cover him. To prevent that we use clear.

This float technique allows us to construct, for example, two column structures.

Another aspect which is a pain in the ass, hard to understand and apply is the relative and absolute element positioning. As you can see, large names means dificult. Both positioning is declared by saying position: (relative or absolute). Notice that html is relative.

  • Relative positioning: First declare that position is relative, then when we specifie top, left, right, or bottom. We move the box by these especifications and these especfications are respect the original position of the box. Notice that when we move the box, the element is still in the original flow of the html, so we can say that he's orginal position is still ocupied by himself.
  • Absolute positioning: First declare that position is absolute, then when we specifie top, left, right, or bottom. The only diference with the one whose above is that the movements we specifie are respect the container, the parent tag. If the parent container has a unespecified position, then the box will keep looking up for a especifie container. At last, the html, because it is naturally relative. Notice that when we offset the container the content is also ofssed.


When it comes to Media Queries, these things are used to adapt the site to diferent devices, to create a responsive website. The syntax is the following.

@media (min-width or max-width) { style }. (We can use logic and to link properties max AND min). Media queries works like if statements.

It's important to not overlap the breakpoints. Notice that we usually have to provide a base style.

The reponsive design is based on columns, so is the boostrap that we will be using. It's common to have or divide our website in 12 columns. This way we can distribute diferent boxes width percentatge using media queries to adapt diferent divices. This will be the basics for Boostrap. Boostraps uses these 12 column structure to make a reponsive website.

We wont be including boostrap yet, but is worth knowing how columns in boostrap works. Later in module 3 we will try to get boostrap involved.


The columns in boostrap have the following syntanx:

col-SIZE-SPAN
  • The size stands for size of the device, it can be xs, sm, md, lg.
  • The span stands for the number of columns that the box will occupie, from 1 to 12.

It's worth knowing that for declaring columns we need to declare as parent rows, and before that a container. The row in boostrap is design to aline the text, basically to make the text more pleasant to the eyes. Notice that columns, row, container are classes of Boostrap.
Another thing to know is the fact that if you declare a "xs" size, then the span will always remains intact even we switch devices, in another words, if we want a double columns struct, even we make the browser bigger (use a laptop or simply making bigger the browser) the structure won't change. For the other sizes, if change the size of the browser, they will switch they structure in order to adapt the new size of the browser.
